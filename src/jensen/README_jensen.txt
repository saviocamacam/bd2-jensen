HENRIQUE SOUZA | SAVIO CAMACAM

Descrição

O problema consiste na simulação de um escalonador de transações de um SGBD.
O programa rece do usuário como parâmetros:
* Número de itens de dados;
* Número de transações;
* Número de acessos;

Como opções de menu, apenas opções 1 e 2 são suportadas correspondendo as funções requeridas para essa fase:
Apenas números inteiros devem ser inseridos para garantir sua execução.
Erros para execução de comandos fora de ordem ainda não foram tratados, logo a opção 2 não deve ser executada antes da opção 1
O programa não suporta o resetamento dos arquivos, por uma falha de sincronia dos índices, assim, para que o teste tenha os 
resultados esperados é necessário reexecutar o programa a cada novo teste - os arquivos serão sobrescritos.

A classe principal é a classe JENSEN - nome escolhido para o projeto.
